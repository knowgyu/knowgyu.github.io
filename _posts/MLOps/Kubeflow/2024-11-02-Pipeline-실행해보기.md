## Kubeflow Pipeline 실행 개념

Kubeflow를 사용하려면 **컴포넌트**와 **파이프라인**을 함께 작성해야 합니다.

Pipeline 공식 문서: [https://www.kubeflow.org/docs/components/pipelines/v1/overview/quickstart/](https://www.kubeflow.org/docs/components/pipelines/v1/overview/quickstart/)<br>

## Component Component-Write

**컴포넌트**는 **컴포넌트 콘텐츠**와 **컴포넌트 래퍼**로 구성됩니다.

하나의 컴포넌트는 래퍼를 통해 Kubeflow에 전달되며, 전달된 컴포넌트는 정의된 콘텐츠를 실행하고 아티팩트를 생성합니다.

### Component Contents

컴포넌트 콘텐츠는 크게 3가지 요소로 구성됩니다.

1. Environment
2. Python code with Config
3. Generates Artifacts

예를 들어 데이터를 불러와 SVC를 학습한 뒤 모델을 저장하는 과정은 아래처럼 나눌 수 있습니다.

```python
import dill
import pandas as pd

from sklearn.svm import SVC

train_data = pd.read_csv(train_data_path)
train_target= pd.read_csv(train_target_path)

clf= SVC(
    kernel=kernel
)
clf.fit(train_data)

with open(model_path, mode="wb") as file_writer:
     dill.dump(clf, file_writer)
```

1. Environment

```python
import dill
import pandas as pd

from sklearn.svm import SVC
```

1. Python code with Config

```python
train_data = pd.read_csv(train_data_path)
train_target= pd.read_csv(train_target_path)

clf= SVC(
    **kernel=kernel** <--- Config
)
clf.fit(train_data)
```

1. Generates Artifacts

```python
with open(model_path, mode="wb") as file_writer:
     dill.dump(clf, file_writer)
```

### Component Wrapper

컴포넌트 래퍼는 컴포넌트 콘텐츠에 필요한 Config를 전달하고 실제 실행을 담당합니다.

Kubeflow에서는 이 컴포넌트 래퍼를 **함수 형태**로 작성합니다.

위 컴포넌트 콘텐츠를 래퍼가 감싸면 아래와 같은 구조가 됩니다.

```python
def train_svc_from_csv( ... ):
	import dill
	...생략
	with open(model_path, mode="wb") as file_writer:
	     dill.dump(clf, file_writer)
```

## Artifacts

아티팩트란 evaluation, result, log 등 파일로 생성되는 결과물을 통틀어 가리키는 말입니다.

**Model**

파이썬 코드와 학습된 Weights와 Network 구조, 그리고 실행시키기 위한 환경

**Data**

전 처리된 피처, 모델의 예측 값 등

**Metric**

동적 지표 - train loss와 같이 학습 진행 중 계속 변화하는 값

정적 지표 - 학습이 끝난 후 최종적으로 모델을 평가하는 정확도 등

# Pipeline

파이프라인은 컴포넌트의 집합과, 이를 어떤 순서로 실행할지 나타내는 흐름으로 구성됩니다.

이 흐름은 방향 순환이 없는 그래프(DAG)이며, 간단한 조건문도 포함할 수 있습니다.

![Untitled](/assets/img/kubeflow/kubepipe501.png)

## Pipeline Config

컴포넌트를 실행하려면 Config가 필요합니다.
파이프라인을 구성하는 여러 컴포넌트의 Config를 모아 둔 것이 파이프라인 Config입니다.

## Run

파이프라인이 필요로 하는 Config가 주어져야만 파이프라인을 실행할 수 있습니다.

Kubeflow에서는 이렇게 실행된 파이프라인을 `Run` 이라고 부릅니다.

파이프라인이 실행되면 각 컴포넌트가 아티팩트를 생성합니다.

Kubeflow Pipelines는 Run마다 고유한 ID를 생성하고, 해당 Run에서 생성된 아티팩트를 함께 저장합니다.
